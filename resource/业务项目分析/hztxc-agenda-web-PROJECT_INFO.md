# HZTX Agenda Web 项目文档

## 1. 项目概况

### 1.1 基本信息

**项目名称**: hztxc-agenda-web  
**项目版本**: 0.0.1  
**项目类型**: 后台管理系统  
**技术栈**: 
- 前端框架: Vue 2.6.10
- UI 组件库: Element UI (@szhn/element-ui 2.15.17)
- 状态管理: Vuex 3.1.0
- 路由管理: Vue Router 3.0.2
- HTTP 客户端: Axios 0.18.1
- 日期处理: Day.js 1.8.36
- 国际化: Vue I18n 7.3.2
- 认证: UAA (@szhn/uaa-design-vue 1.0.5)

**项目初始化时间**: 2024年2月  
**当前分支**: dev

### 1.3 主要功能模块

1. **待办事项管理**
   - 待办列表展示
   - 待办创建、编辑、查看详情
   - 待办完成、取消、重启
   - 待办催办功能
   - 待办分享功能

2. **用户管理**
   - 执行人管理（添加、标记、移除）
   - 参与人管理
   - 通知管理

3. **文件管理**
   - 附件上传
   - 文件预览
   - 文件下载

4. **评论系统**
   - 待办详情评论
   - 评论区样式优化

5. **标签管理**
   - 标签创建、选择、删除
   - 标签搜索
   - 标签颜色枚举

6. **海政通集成**
   - 海政通登录
   - 海政通选人组件
   - 海政通 ticket 获取

7. **系统管理**
   - 权限控制
   - 国际化支持
   - 响应式设计
   - 火土熊监控集成

## 2. 项目亮点

### 2.1 技术亮点

1. **基于成熟框架**：基于 vue-element-admin 模板，具有完善的后台管理功能体系
2. **组件化开发**：高度模块化的组件设计，提高代码复用性
3. **海政通集成**：深度集成海政通登录和选人功能，符合企业内部系统需求
4. **响应式设计**：适配不同屏幕尺寸，提供良好的用户体验
5. **国际化支持**：内置多语言切换功能，支持中英文
6. **权限管理**：完善的权限控制体系，基于 UAA 认证
7. **性能优化**：按需导入组件，减少打包体积
8. **监控集成**：集成火土熊监控，实时监控系统运行状态

### 2.2 功能亮点

1. **完整的待办管理流程**：从创建到完成的全流程管理，支持多种操作
2. **丰富的通知方式**：支持多种通知方式，确保待办及时处理
3. **文件管理功能**：支持附件上传、预览和下载，方便文件共享
4. **评论系统**：支持待办详情评论，促进沟通协作
5. **标签管理**：支持标签分类和搜索，提高待办管理效率
6. **智能时间处理**：最近时间格式化，提供更直观的时间显示
7. **外部链接支持**：支持外部创建待办链接，扩展系统使用场景
8. **用户友好界面**：精心设计的 UI 界面，提供良好的用户体验

## 3. 已解决和优化的问题

### 3.1 功能问题

1. **待办详情参与人添加人员时数据返回不一致问题**
   - 原因：接口返回数据格式与前端期望不一致
   - 解决方案：调整前端数据处理逻辑，适配接口返回格式

2. **附件文件名称乱码问题**
   - 原因：文件下载时响应类型设置不正确
   - 解决方案：修改文件下载响应类型，确保文件名正确显示

3. **标签组件无列表数据使用占空显示**
   - 原因：标签组件在无数据时没有合适的占位显示
   - 解决方案：添加占空显示逻辑，提高用户体验

4. **新建待办默认选中所有通知方式**
   - 原因：创建待办时通知方式默认值设置不合理
   - 解决方案：修改默认值逻辑，确保通知方式正确设置

5. **标签列表刷新不及时问题**
   - 原因：标签操作后列表没有及时刷新
   - 解决方案：优化标签操作后的刷新逻辑

6. **评论区样式问题**
   - 原因：评论区样式设计不合理
   - 解决方案：调整评论区样式，提高可读性

7. **选人组件参数和回显逻辑问题**
   - 原因：海政通选人组件参数设置和回显逻辑有误
   - 解决方案：修改选人组件参数和回显逻辑

8. **通知类型复制错误问题**
   - 原因：通知类型复制逻辑有误
   - 解决方案：修复通知类型复制功能

9. **创建待办之后重新创建原来创建的值没有重置**
   - 原因：表单重置逻辑不完善
   - 解决方案：优化表单重置逻辑

10. **海政通选入赋值逻辑问题**
    - 原因：海政通选人后赋值逻辑有误
    - 解决方案：修改选入赋值逻辑

### 3.2 技术问题

1. **海政通获取 ticket 方法响应取值变换处理**
   - 原因：海政通 API 响应格式变更
   - 解决方案：调整 ticket 获取方法，适配新的响应格式

2. **按需导入组件依赖**
   - 原因：组件依赖导入方式不合理，导致打包体积过大
   - 解决方案：优化组件导入方式，按需导入

3. **最近日期时间格式化处理**
   - 原因：日期时间显示不够直观
   - 解决方案：实现最近时间格式化功能，提供更友好的时间显示

4. **项目登录 AK 地址编码配置**
   - 原因：登录地址编码配置不正确
   - 解决方案：正确配置登录 AK 地址编码

5. **火熊应用 key 以及加盐校验码配置**
   - 原因：火熊监控配置不完整
   - 解决方案：完善火熊应用配置

6. **二级目录配置**
   - 原因：项目部署路径配置不合理
   - 解决方案：添加二级目录配置支持

7. **UAA 选人组件包依赖安装**
   - 原因：UAA 选人组件依赖缺失
   - 解决方案：安装并配置 UAA 选人组件依赖

8. **项目根路径路由地址配置**
   - 原因：路由地址配置不正确
   - 解决方案：修改项目根路径路由配置

9. **新建代办弹窗显示时机处理**
   - 原因：弹窗显示时机控制不合理
   - 解决方案：优化弹窗显示逻辑

10. **接口调用获取 Uaa 选人组件认证配置**
    - 原因：UAA 认证配置获取方式有误
    - 解决方案：修复 UAA 认证配置获取逻辑

## 6. 项目总结

HZTX Agenda Web 项目是一个基于 Vue 2.6.10 和 Element UI 开发的后台管理系统，主要功能是待办事项管理。项目集成了海政通登录和选人功能，具有完整的待办管理流程、文件管理、评论系统和标签管理等功能。

项目的技术亮点包括组件化开发、响应式设计、国际化支持、权限管理和性能优化等。通过解决一系列功能和技术问题，项目的稳定性和用户体验得到了显著提升。

杜审言作为主要开发者之一，在待办详情功能、标签管理、海政通集成和技术优化等方面做出了重要贡献。

项目采用了规范的 Git 提交流程，遵循 Conventional Commits 规范，确保代码提交的清晰性和可追踪性。
